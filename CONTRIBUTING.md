# 贡献指南

感谢你对本项目感兴趣。

## 关于 Pull Request

> [!IMPORTANT]
> **本仓库目前不接受外部 Pull Request。**

本项目以个人维护为主，代码方向、架构选型、发布节奏都由维护者把关。直接合并外部 PR 会带来审查与维护成本，因此默认所有未经事先沟通的 PR 都会被关闭。

如果你有想法或改动建议，请改用下面两种方式：

### 1. 提 Issue 讨论

无论是 bug、功能建议、还是「我想顺手提个 PR 修一下」，都请先开 [Issue](../../issues) 描述清楚：

- **Bug**：复现步骤、期望行为、实际行为、版本号、运行环境
- **功能建议**：你想解决什么问题，为什么现有方式不够，期望的形态
- **文档改进**：哪一段不清楚，建议怎么改

维护者评估后会决定是否纳入计划。

### 2. Fork 自用

本项目以 [MIT License](LICENSE) 开源，欢迎 fork 后按你的需求自行修改、部署、二次开发。请保留版权声明。

> [!CAUTION]
> Fork 自用时请遵守 [README](README.md) 中的免责声明，本项目涉及对 ChatGPT 官网接口的逆向研究，**仅供个人学习与非商业性技术交流**，严禁用于任何商业用途、批量自动化滥用、套利倒卖或违法用途。

## 关于 Issue

为了让问题能被快速定位，请：

- **搜一下**已有 Issue，避免重复
- 标题写清楚现象，不要只写「不能用」「报错了」
- 贴日志时使用代码块（` ``` `），并去掉 `auth-key`、`access_token`、邮箱密码等敏感信息
- 涉及 Docker 部署问题，附上 `docker compose` 版本、镜像 tag、`docker logs` 关键片段

## 安全问题

发现安全漏洞请**不要**直接开公开 Issue，通过 GitHub 的 [Security Advisories](../../security/advisories) 功能私下报告，或在 Issue 中只提及「发现安全问题，请联系我」由维护者安排私下沟通渠道。
